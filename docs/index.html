Capabilities

Everything a metering surface should be

Built from scratch for REAPER power users who want real hardware feedback — not another plugin window.

📊
Real-Time LUFS Metering
Momentary, Short-term, and Integrated LUFS with configurable loudness targets. Streaming (−14 LUFS), broadcast (−23 LUFS), podcast, film — one tap to switch.
🎚
RMS & True Peak
Dual-channel RMS and True Peak metering with configurable peak hold ballistics. Session MAX readout with color-coded over-threshold alerts.
🌈
16-Band Spectrum Analyzer
Complete 16-band FFT spectrum from 31 Hz to 16 kHz, powered by a custom JSFX plugin. Gradient gradient color modes: Classic, Neon, Fire, Rainbow.
🔄
Phase Correlation
Real-time stereo phase correlation meter (-1 to +1). Instant visual warning when your mix is approaching mono compatibility issues.
🎛
Full Transport Control
Play, Stop, Record, Rewind, Fast Forward — all from the touchscreen. Navigate between markers, toggle track mute/solo/arm without touching your keyboard.
🖥
Portrait & Landscape
Rotate the display 90° and the entire UI reflows. Portrait mode shows compact track info with large LUFS readouts. Landscape mode maximizes meter height.
🎨
5 Professional Themes
Codeine Crazy (deep purple), Terminal Green, Arctic Blue, Midnight Amber, and Graphite. All designed for studio ambient light conditions.
📡
WiFi + USB Dual Mode
Connect wirelessly via UDP (sub-50ms latency) or over USB serial for zero-WiFi setups. Automatic fallback. WebSocket-free simplicity.
🔧
OTA Firmware Updates
Update firmware wirelessly over WiFi — no USB cable needed after initial flash. Progress is shown live on the display.
Architecture

How MeterBridge works

A clean four-layer signal path from REAPER's audio engine to your physical display.

1
JSFX Plugin
A custom JSFX effect on your master bus performs real-time FFT analysis, LUFS integration, and RMS/peak tracking. Zero CPU overhead.
2
Lua Bridge
A lightweight REAPER Lua script reads audio metrics and transport state every 20–50 ms, serializes them to JSON, and hands off to the relay.
3
Python Relay
The relay bridges REAPER to the ESP32 — either via UDP over WiFi (<5ms) or USB serial. It also receives transport commands going the other way.
4
ESP32 Display
The ESP32-S3 renders all meters, spectrum, and transport state on the 7" touch display at up to 240 FPS using LVGL with GPU-accelerated drawing.
Requirements

What you need

Designed to be as accessible as possible. If you run REAPER, you're 95% of the way there.

🖥 Hardware
  • Recommended hardware: Elecrow CrowPanel 7" ESP32-S3 HMI Display — the exact board this project was developed and tested on.
  • Any ESP32-S3 with 7" RGB touch LCD (800×480) and GT911 touch controller
  • USB-C cable for initial flash
  • WiFi 2.4 GHz network (optional — USB works too)
  • Power supply via USB (5V/1A minimum)
🎛 Software
  • REAPER DAW (any recent version)
  • Python 3.8+ (for the relay)
  • PlatformIO (for firmware build — included in script)
  • Windows 10/11 (macOS/Linux with minor adjustments)
  • SWS Extension for REAPER (recommended)
💰 Estimated Cost
  • CrowPanel 7" board: ~$35–50 USD
  • USB-C cable: ~$5
  • Software: 100% free & open source
  • Total: ~$40–55 USD

Installation

Up and running in minutes

The entire setup is driven by batch scripts — no command line needed.

1
Download & extract MeterBridge
Clone the repository or download the ZIP from GitHub. Extract anywhere — no spaces in the path preferred (e.g. C:\meterbridge).
git clone https://github.com/s0wingseason/ESP32-Touch-Display-REAPER-Advanced-Metering-and-Transport-Controller.git
2
Connect your ESP32 via USB and run build_and_run.bat
Double-click build_and_run.bat in the root folder. It auto-installs PlatformIO, builds the firmware, detects your COM port, and flashes the ESP32. Then go grab a coffee ☕
build_and_run.bat ← just double-click this
3
Install the REAPER bridge scripts
Double-click install_reaper_bridge.bat. This copies the Lua script and JSFX plugin to your REAPER scripts folder automatically.
4
Configure WiFi on the display
On first boot, MeterBridge creates a WiFi AP called MeterBridge-Setup. Connect to it from your phone or laptop, select your network, enter the password. Done.
5
Start the relay and open REAPER
Double-click run_relay.bat. Then in REAPER, add the MeterBridge JSFX plugin to your master track and run the Lua bridge script via the Actions menu. Meters appear instantly.
run_relay.bat ← runs the Python bridge
Troubleshooting

Known issues & fixes

Every problem we've seen in the wild, documented with a clear fix.

The display stays black after flashing +
This almost always means the firmware was built for the wrong board variant. Ensure you're using the crowpanel_7inch PlatformIO environment. Also try holding the BOOT button on the board, then pressing RESET before clicking upload. Some boards need manual boot mode entry.
Meters aren't receiving data / connection light is red +
Check that: (1) The relay script (run_relay.bat) is running, (2) the JSFX plugin is on the master track and active, (3) the Lua bridge script is running in REAPER (Actions → Show action list → find MeterBridge and run it), (4) the ESP32 and your PC are on the same WiFi network. The display shows its IP address in the status bar — ping it from your PC to verify connectivity.
WiFi won't connect / keeps falling back to AP mode +
The ESP32 only supports 2.4 GHz WiFi — 5 GHz networks will never appear in the scan. Also confirm the password has no special characters that might encode incorrectly. If the SSID contains spaces or Unicode characters, try a simpler hotspot name. You can reset WiFi credentials by going to Settings → WiFi on the display, or by sending NVS_CLEAR over USB serial.
Frame rate is low / UI feels sluggish +
Ensure PSRAM draw buffers allocated successfully — check the serial log for [DISP] 200-row PSRAM draw buffers allocated (full speed). If you see the fallback warning, the board may have a different PSRAM configuration. Also try reducing the update rate in Settings from 50 ms to 100 ms. For OPI PSRAM boards, confirm board_build.arduino.memory_type = qio_opi in platformio.ini.
Touch input is inaccurate or not registering +
Most CrowPanel 7" boards use a GT911 capacitive touch controller over I2C. If touch seems offset, try rotating the display 180° in Settings → Display → Rotation. For V3.0 boards with the PCA9557 IO expander, ensure the firmware detected it on startup (look for [HW] PCA9557: FOUND (V3.0) in serial output).
OTA firmware update fails +
OTA is considered experimental. If it fails, use the wired USB method: run flash_esp32.bat or run pio run -e crowpanel_7inch --target upload from the esp32-firmware directory. OTA failures are usually caused by network instability during the large ~1 MB transfer.
JSFX plugin isn't doing anything / spectrum is flat +
Confirm the JSFX plugin (meterbridge_spectrum.jsfx) is installed in your REAPER Effects folder and appears in the FX chain of your master bus. It must be before any limiting/clipping at the master bus level to get accurate readings. Also ensure the plugin is not bypassed (green checkbox in the FX chain).
